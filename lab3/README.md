## 1. Діаграма “Creating Page” — створення сторінки редактором

Дана діаграма описує процес створення нової сторінки користувачем-редактором. Процес складається з низки послідовних кроків, що включають ініціалізацію, наповнення сторінки елементами та фіналізацію.

Учасники процесу

Editor — користувач, який створює сторінку.

Creating Page — модуль логіки створення сторінки.

DB — база даних, що зберігає структуру, контент та метадані сторінки.

Опис послідовності

Створення сторінки:
Редактор надсилає команду createPage(), після чого модуль створення сторінки ініціює запис нової сторінки до бази даних.
База повертає відповідь про успішність операції.

Додавання структури:
Редактор викликає addingStructure().
Структура сторінки записується до бази даних, після чого надсилається відповідь про успішне збереження.

Додавання тексту:
Через запит addingText() редактор передає текстовий контент.
Модуль відправляє дані до бази, яка підтверджує збереження.

Додавання зображень:
Команда addingPhoto() ініціює передавання зображень у систему.
Файли зберігаються у базі даних, після чого редактор отримує відповідь.

Фіналізація:
Команда Finish() завершує роботу зі сторінкою.
Модуль виконує фінальне збереження в БД і повертає підтвердження редактору.

## 2. Діаграма “Taking Page” — отримання сторінки користувачем

Ця діаграма описує процес отримання вже створеної сторінки користувачем. Процес полягає у поетапному запиті частин сторінки: структури, тексту та зображень.

Учасники процесу

User — користувач, який переглядає сторінку.

Taking Page — модуль, що відповідає за завантаження даних сторінки.

DB — база даних, що зберігає весь контент.

Опис послідовності

Запит сторінки:
Користувач викликає TakingPage(), після чого модуль звертається до бази даних, отримує основну інформацію про сторінку й повертає її користувачу.

Отримання структури:
Запит takingStructure() використовується для завантаження структурної частини сторінки.
Модуль отримує структуру з бази та передає її користувачу.

Отримання тексту:
Запит takingText() ініціює отримання текстового контенту зі сторінки.
Після отримання з бази модуль повертає текст користувачу.

Отримання зображень:
Через команду takingPhoto() користувач може завантажити зображення сторінки.
Модуль звертається до бази, отримує медіафайли і передає їх користувачу.
