#Текстовий опис сценарію
1. Поки турнір триває

Користувач регулярно переглядає сторінку турніру → Take page

Адміністратор створює турнір → Make tour

Адміністратор оновлює інформацію турніру → Update tour

Цикл продовжується, поки турнір не завершено.

2. Завершення турніру

Адміністратор завершує турнір → Finish tour

Liquidpedia повертає оновлену сторінку користувачу → Take page

3. Створення сторінок користувачів після турніру

Користувач переглядає сторінку → Take page

Адміністратор створює сторінку користувача → Make user page

Liquidpedia повертає оновлену сторінку → Take page

4. Подальше оновлення

Адміністратор робить оновлення → Update

Адміністратор створює ще одну сторінку → Make user page

Користувач знову переглядає → Take page

5. Додавання призів

Адміністратор вносить чергове оновлення → Update

Додає приз → Add prize
