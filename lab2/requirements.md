# Вимоги до системи управління кіберспортивними турнірами

## 1. Функціональні вимоги (Functional Requirements)

| № | Опис вимоги |
|---|--------------|
| **FR1** | Система повинна дозволяти організатору створювати новий турнір із назвою, датою та призовим фондом. |
| **FR2** | Організатор може додавати матчі до турніру. |
| **FR3** | Організатор може вносити результати матчів. |
| **FR4** | Організатор може реєструвати команди та їхні склади. |
| **FR5** | Система повинна дозволяти зберігати командні та індивідуальні призові. |
| **FR6** | Команди можуть переглядати історію своїх виступів. |
| **FR7** | Гравці можуть переглядати власну історію виступів і призові. |
| **FR8** | Вболівальники можуть переглядати турніри, результати та статистику. |
| **FR9** | Система повинна інтегруватися з аналітичними сервісами для збору статистики. |

---

## 2. Нефункціональні вимоги (Non-Functional Requirements)

| № | Вимога |
|---|---------|
| **NFR1** | Система має забезпечувати цілісність даних (неперетин складів, валідація дат). |
| **NFR2** | Інтерфейс повинен бути доступним для всіх ролей користувачів. |
| **NFR3** | Операції створення турніру, матчів і результатів мають виконуватися менше ніж за 3 секунди. |
| **NFR4** | Дані мають бути збережені у базі даних із підтримкою транзакцій. |
| **NFR5** | Система має забезпечувати інтеграцію з REST API для аналітичних сервісів. |

---

## 3. Сторони, зацікавлені у системі

- **Організатор турніру** — створює, редагує й адмініструє турніри.  
- **Команда** — реєструє склад і переглядає результати.  
- **Гравець** — переглядає власні призові та історію участі.  
- **Вболівальник / Аналітик** — спостерігає за турнірами, переглядає статистику.  
- **Зовнішня аналітична система** — отримує дані через API для подальшої обробки.
